## 二、开发流程中的强制审查点

```
需求确认 → 设计评审 → 开发 → 自测 → 代码审查 → 集成测试 → 部署
                              ↑          ↑          ↑
                           审查点1   审查点2   审查点3
```

### 强制审查节点

| 节点         | 时机           | 审查内容               | 通过标准           |
| ------------ | -------------- | ---------------------- | ------------------ |
| **开发自测** | 功能开发完成后 | 功能完整性、基本可用性 | 核心功能可运行     |
| **代码审查** | 提交审查前     | 代码质量、规范、安全   | 审查清单通过率≥90% |
| **集成测试** | 合并主干前     | API联调、端到端测试    | 测试用例100%通过   |
